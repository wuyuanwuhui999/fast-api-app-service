# 项目背景
我有一个 FastAPI 多模块项目，结构如下：

## 模块列表
- common（共用模块，包含通用工具、配置、数据库连接等）
- user（用户模块）
- chat（聊天模块）
- prompt（提示词模块）
- tenant（租户模块）
- gateway（网关模块）

## 模块识别规则
URL 格式：`/service/{模块名}/...`
- `/service/user/getUserData` → user 模块
- `/service/prompt/getPrompt` → prompt 模块

## 模块内部结构
每个业务模块（user/chat/prompt/tenant/gateway）均包含：
模块名/
├── dependencies/ # 依赖注入、权限校验
├── models/ # 数据库模型（SQLAlchemy等）
├── repositories/ # 数据访问层（CRUD操作）
├── routers/ # 路由定义（API端点）
├── schemas/ # Pydantic 请求/响应模型
└── service/ # 业务逻辑层


## 架构特点
- 所有服务注册到 Nacos
- 网关模块负责路由转发和请求日志（异步写入数据库）
