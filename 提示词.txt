# 项目背景
我有一个 FastAPI 多模块项目，包含以下 5 个独立可运行的模块：
- common（共用模块）
- user（用户模块）
- chat（聊天模块）
- prompt（提示词模块）
- tenant（租户模块）
- gateway（网关模块）

# 模块识别规则
根据接口 URL 中的路径段可判断所属模块：
- URL 格式：`/service/{模块名}/...`
- 示例：
  - `/service/user/getUserData` → `user` 模块
  - `/service/prompt/getPrompt` → `prompt` 模块

# 模块内部结构
每个业务模块（user / chat / prompt / tenant / gateway）均包含以下标准目录：

dependencies/       # 依赖注入或权限校验等
models/             # 独享的数据库模型文件
repositories/       # 数据库读写操作
routers/            # 路由定义
schemas/            # Pydantic 参数模型
service/            # 业务逻辑层
